## 2. Packet format (`packet.py`)

Header (struct format `'!IIbIB'`, 14 bytes, network byte order):

| Field | Type | Size | Meaning |
|---|---|---|---|
| `seq_num` | `I` | 4 bytes | sequence number |
| `ack_num` | `I` | 4 bytes | ack number (only meaningful on ACK packets) |
| `ptype` | `b` | 1 byte | packet type, see below |
| `payload_len` | `I` | 4 bytes | length of payload in bytes |
| `chksum` | `B` | 1 byte | 8-bit checksum, computed last |

**Types** (`Packet.TYPE_*`): `DATA=0`, `ACK=1`, `START=2`, `END=3`.

- `START` = "notify receiver a file is coming" (assignment's "upload a file")
- `DATA` = a `CHUNK_SIZE=1024`-byte slice of file content
- `END` = "completion of file transfer" notice
- `ACK` = acknowledgment, `ack_num` field carries the seq being acked

**Checksum:** sum of all header bytes (minus `chksum` itself) + all payload bytes, folded into 8 bits by repeatedly adding the carry back in, then XORed with `0xFF`. This is what's used for the bonus corruption-detection feature ([Section 4](#4-bonus-work-already-done)). `Packet.from_bytes()` raises `ValueError` on mismatch; every receiver/sender treats that as "discard, don't ACK, let the timeout handle it" — never crashes the process.

**File name convention:** sender's `check_file_name()` requires exactly one `.` in the filename (e.g. `file_10KB.bin`). Extension-less or multi-dot names typed at the CLI prompt will raise `InvalidFileName`.

## 3. How to run a single manual transfer

### Stop-and-Wait (receiver listens on UDP port 6969)

```
Terminal 1:  python receiver_stopwait.py --loss-rate 0.1 --corruption-rate 0.0
Terminal 2:  python sender_stopwait.py
             Enter the file name to be transferred (with extension): myfile.txt
```

(Run sender from a directory that actually contains `myfile.txt`; the received copy is written by the receiver into **its** current directory.)

### Go-Back-N (receiver listens on UDP port 6970)

```
Terminal 1:  python receiver_gbn.py --loss-rate 0.1 --corruption-rate 0.0
Terminal 2:  python sender_gbn.py
             Enter the file name to be transferred (with extension): myfile.txt
```

### Command-line flags, by script

**`receiver_stopwait.py` / `receiver_gbn.py`**

| Flag | Default | Meaning |
|---|---|---|
| `--loss-rate FLOAT` | `0.0` | Probability [0.0-1.0] of dropping each incoming packet before parsing. |
| `--corruption-rate FLOAT` | `0.0` | Probability [0.0-1.0] of flipping one random bit in each incoming packet before parsing (after the loss check). |

**`sender_stopwait.py` / `sender_gbn.py`**

| Flag | Default | Meaning |
|---|---|---|
| `--loss-rate FLOAT` | `0.0` | Accepted but unused — kept only so the sender's CLI matches the receiver's. Loss is simulated receiver-side only. |
| `--corruption-rate FLOAT` | `0.0` | Probability [0.0-1.0] of flipping one random bit in each incoming ACK before parsing. |

**`run_experiments.py`**

| Flag | Meaning |
|---|---|
| `--quick` | Flag, no value. Runs a small smoke-test matrix (2 sizes, 2 loss rates, 1 trial) to confirm the pipeline works, instead of the full report-ready matrix. |
| `--timeout FLOAT` (default `120.0`) | Max seconds to let a single sender subprocess run before it's killed and the trial recorded as failed. Safety net, not part of the protocol — see earlier discussion on why this shouldn't just be set huge. |
| `--skip-bonus` | Flag, no value. Skips the corruption-rate matrix, runs only the loss-rate matrix. |

**`visualize.py`** — no flags, always reads `results.csv` and writes `summary.csv` + `plots/`.

### Notes / assumptions baked into the code

- Both client and server assumed to be on localhost; ports are hardcoded (6969 stopwait, 6970 GBN) at the top of each script — change there if you need to run sender/receiver on different machines.
- On localhost RTT is near-zero, which removes the main thing GBN's pipelining is designed to hide — at 0% loss the two protocols can end up close, with the gap mostly coming down to per-packet overhead (GBN's sender does a `select()` ACK-poll plus window bookkeeping per packet; Stop-and-Wait just blocks on `recvfrom()`). Worth keeping in mind if the 0%-loss numbers look closer than expected — that's the environment, not a bug.
- `--loss-rate` is parsed by BOTH sender and receiver CLIs for uniformity, but only the RECEIVER actually drops incoming packets. Passing `--loss-rate` to the sender is a no-op (documented in its own `--help` text) — there's no sender-side loss to find, it's intentionally receiver-only.
- `--corruption-rate` IS active on both ends: receivers corrupt incoming DATA/START/END packets before parsing them; senders corrupt incoming ACK packets before parsing them. So corruption is simulated bidirectionally, loss is simulated receiver-side only.
- `TIMEOUT = 0.3s` and `CHUNK_SIZE = 1024` bytes on both protocols. `WINDOW_SIZE = 8` for GBN (hardcoded in `sender_gbn.py`).
- Stop-and-Wait retries forever on timeout (no retry cap) — by design, so it can complete under sustained loss, matching GBN's behavior.
- Sequence numbers: Stop-and-Wait uses 1-bit alternating (0/1) sequence numbers, classic alternating-bit protocol. GBN uses unbounded incrementing integers (no modulo wraparound — fine for the file sizes in this assignment, would need revisiting for very long transfers).

## 4. Bonus work already done

We implemented **Bonus Option B** (packet corruption simulation, +5%), not Option A (Selective Repeat). Worth flagging since the assignment treats them as alternatives rather than additive — if anyone's tempted to also add Selective Repeat for extra credit, just know it'd be on top of B, not required.

**What's implemented for Option B:**

- 8-bit checksum field in every packet (`packet.py`, [Section 2](#2-packet-format-packetpy) above).
- `--corruption-rate` flag on all 4 scripts: with that probability, a single random bit in a single random payload/header byte (excluding the checksum byte itself) is flipped before the packet is handed to `Packet.from_bytes()`. This is checked on receivers (incoming DATA/START/END) and on senders (incoming ACK).
- Corrupted packets fail the checksum check and are silently dropped (no ACK sent), relying on the existing timeout/retransmit machinery — no separate corruption-handling code path was needed.
- `run_experiments.py` / `visualize.py` already have a second experiment matrix and plot (`plot_corruption.png`) dedicated to corruption rates `[0.0, 0.05, 0.10, 0.15]`, parallel to the loss-rate matrix.

`Bonus.txt` (required submission file) has **NOT** been written yet — it just needs to describe this in a paragraph or two for the marker. The technical work is done; only the writeup is missing.
